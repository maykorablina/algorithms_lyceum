
## Что такое функция?

  

Функция — это «маленькая программа внутри программы». Она выполняет определённое действие и может возвращать результат.

  

## Зачем нужны функции?

  

- Чтобы не дублировать код (написал один раз — вызывай много раз).

- Чтобы разделять программу на части (удобнее понимать).

- Чтобы делать код короче и понятнее.

  

## Общий вид функции

  

```plaintext
тип_возврата имя_функции(список_параметров) {
    // тело функции — что она делает
    return значение; // (если тип_возврата не void)
}
```

  

## Пример функции

  

```cpp
// Функция складывает два числа и возвращает результат
int add(int a, int b) {
    return a + b;
}
```

  

### Вызов функции:

  

```cpp
int result = add(5, 7); // result = 12
```

  

## Основные моменты

  

### Тип возврата

  

-  `int`, `double`, `char` и т. д. → функция возвращает значение этого типа.

-  `void` → функция ничего не возвращает.

  

```cpp
void hello() {
    std::cout << "Привет!\n";
}
```

  

### Параметры

  

Функции можно передавать данные:

  

```cpp
int square(int x) {
    return x * x;
}

```

  

### Вызов:

  

```cpp
std::cout << square(5); // 25
```

  

Можно несколько параметров:

  

```cpp
double average(int a, int b) {
    return (a + b) / 2.0;
}
```

  

### Возврат результата (return)

  

Функция может вернуть значение. После `return` выполнение функции останавливается. У функции типа `void`  `return` писать не обязательно.

  

## Где объявлять функции?

  

Функцию нужно объявить до `main`. Причем определение можно написать и после `main`, но объявить надо обязательно!

  

### Пример с прототипом:

  

```cpp
#include <iostream>

// объявление
int square(int x);

int main() {
	
    std::cout << square(6);
    return 0;
}

// определение (тело)
int square(int x) {
    return x * x;
}
```

  

### One definition rule

  

В C++ есть важное правило: **каждая функция в программе должна иметь только одно определение** (тело).

Объявлений может быть сколько угодно, но само тело функции должно встречаться**&#32;только один раз** во всей программе.

  

Если написать два разных определения, получится ошибка «multiple definition

  

**Пример:**

  

```cpp
int add(int a, int b) { return a + b; }
int add(int a, int b) { return a - b; } // ошибка
```
